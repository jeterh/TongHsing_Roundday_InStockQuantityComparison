全日貨運 / 同興實業 庫存比對工具

連結：https://jeterh.github.io/TongHsing_Roundday_InStockQuantityComparison/
 
本文件記載了「全日庫存表」與「同興庫存表」比對工具的開發重點與使用注意事項。

🚀 版本發布紀錄 (Release Notes)

v1.0.0 (2026-02-14) - 初始版本發布

核心功能：

支援 Excel 檔案 (.xlsx, .xls) 上傳。

全日庫存表解析：自動讀取第一列作為標題，支援「貨號」與「庫存數量」欄位 Group By 加總。

同興庫存表解析：自動偵測第 7-8 列的標題列，支援「貨品代號」與「副單位數量」欄位 Group By 加總。

智慧比對邏輯：

以「貨號 = 貨品代號」為基準。

找出相同編號但數量不一致（全日庫存 vs 同興副單位數量）的品項。

UI 特色：

類似 Excel 的格狀化比對結果表格。

狀態標記功能（數量不符、全日缺漏、同興缺漏）。

響應式設計，支援拖曳上傳。

部署技術：

基於 React + Vite 開發。

使用 Tailwind CSS 進行視覺設計。

透過 GitHub Actions 自動部署至 GitHub Pages。

💡 使用提醒 (Reminders)

1. 檔案格式要求

全日庫存表：

必須包含標題列「貨號」與「庫存數量」。

系統預設讀取檔案內的第一個工作表 (Sheet 1)。

同興庫存表：

必須包含標題列「貨品代號」與「副單位數量」。

系統會自動搜尋前 15 列，尋找標題起始位置（通常在第 7 或 8 列）。

2. 比對邏輯說明

資料清洗：系統會自動去除貨號前後的空白字元。

加總機制：如果同一個 Excel 內有重複的貨號，系統會先進行自動加總後再與另一份表格比對。

差異定義：

數量不符：兩份表格都有此貨號，但加總後的庫存數值不同。

全日缺漏：同興表有此貨號，但全日表完全找不到。

同興缺漏：全日表有此貨號，但同興表完全找不到。

3. 系統限制

效能建議：建議單個 Excel 檔案資料量在 10,000 筆以內，以獲得最佳瀏覽器處理速度。

網路環境：本工具會透過 CDN 載入 Excel 處理套件 (xlsx.js)，執行時請確保網路連線正常。

資安保護：所有的檔案解析與比對皆在您的瀏覽器本地端完成，檔案不會上傳至任何伺服器，請安心使用。

🛠️ 維護與技術支援

若遇到標題列無法正確偵測，請檢查 Excel 內的欄位名稱是否與規格（貨號、貨品代號等）字樣完全一致。
